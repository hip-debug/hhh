# Анализ изменений между ветками проекта SFMShop

## Сравнение master vs feature/add-logging

Ветка feature/add-logging добавляет:
- Новый файл: src/utils/logger.py
- Реализация системы логирования с использованием стандартного модуля logging
- Функция setup_logger() создаёт логгер с двумя обработчиками:
  * FileHandler — запись в файл logs/<name>_<дата>.log
  * StreamHandler — вывод в консоль
- Единый экземпляр app_logger для всего приложения

Также включает изменения из рабочей области (src/database/*), 
добавленные перед созданием коммита.

## Сравнение master vs feature/add-caching

Ветка feature/add-caching добавляет:
- Новый файл: src/utils/cache.py
- Реализация кэша в памяти (in-memory cache) с поддержкой TTL
- Класс Cache с методами: get(), set(), delete(), clear()
- Декоратор @cached(ttl) для автоматического кэширования результатов функций
- Глобальный экземпляр cache с TTL по умолчанию 300 секунд

## Выводы

Обе ветки независимы друг от друга и могут быть смерджены без конфликтов.
feature/add-logging затрагивает src/utils/logger.py
feature/add-caching затрагивает src/utils/cache.py
Файлы не пересекаются — параллельная разработка прошла успешно.

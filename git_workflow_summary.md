# Git Workflow для проекта SFMShop

## Использованные команды Git

- `git init` — инициализация репозитория
- `git add <file>` — добавление файлов в staging area
- `git commit -m "message"` — создание коммита с сообщением
- `git status` — просмотр текущего состояния рабочей директории
- `git log --oneline --graph --all` — визуализация истории коммитов всех веток
- `git log --stat` — статистика изменённых файлов по коммитам
- `git branch -a` — список локальных и удалённых веток
- `git checkout -b <branch>` — создание новой ветки и переключение на неё
- `git checkout <branch>` — переключение между ветками
- `git merge <branch>` — слияние ветки в текущую
- `git diff <branch1> <branch2>` — сравнение изменений между ветками
- `git show <commit>` — просмотр конкретного коммита
- `git stash` — временное сохранение незакоммиченных изменений
- `git push origin <branch>` — отправка ветки в удалённый репозиторий

## Созданные ветки

| Ветка | Назначение |
|-------|-----------|
| `master` | Основная ветка проекта, содержит стабильный код |
| `feature/add-shipping` | Добавление метода расчёта стоимости доставки в Product |
| `feature/inventory-managment` | Управление складом: check_stock, update_stock |
| `feature/add-logging` | Система логирования (src/utils/logger.py) |
| `feature/add-caching` | Система кэширования (src/utils/cache.py) |

## Стратегия работы с ветками

### Feature Branch Workflow

Проект использует модель **Feature Branch Workflow**:

1. **master** — защищённая ветка, содержит только проверенный код
2. Каждая новая функциональность разрабатывается в отдельной `feature/*` ветке
3. После завершения feature-ветка сливается в `master` через Pull Request
4. Конфликты разрешаются перед мерджем

### Правила именования веток

```
feature/<краткое-описание-функции>   — новый функционал
fix/<описание-бага>                  — исправление ошибок
hotfix/<критическое-исправление>     — срочные правки в production
```

### Процесс работы над задачей

```
1. git checkout master
2. git pull origin master
3. git checkout -b feature/название-задачи
4. [разработка + коммиты]
5. git push origin feature/название-задачи
6. Создать Pull Request → ревью → merge в master
```

### Соглашения по коммитам

Сообщения коммитов должны быть на русском языке, описывать суть изменений:
- `"Добавлена система логирования"`
- `"Добавлена система кэширования"`
- `"Расширен .gitignore для Python проекта"`
